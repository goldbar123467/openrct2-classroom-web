# Accessibility statement

The Parkworks setup and save-management wrapper targets WCAG 2.2 AA with semantic landmarks, native controls, visible focus, keyboard operation, 200% zoom support, reduced-motion handling, forced-color fallbacks, and high-contrast text.

The OpenRCT2 game itself renders into a canvas and is highly visual. It is not fully compatible with screen readers, and Parkworks does not claim otherwise. Some in-game dialogs and small pixel text may also be difficult for low-vision users even when the browser is zoomed.

Classroom accommodations may include:

- paired play with a student operating and another planning/accounting;
- teacher-provided enlarged screenshots or printed park ledgers;
- a larger external display;
- reduced map size and visual density;
- additional time for track building;
- an equivalent non-canvas theme-park planning activity.

Before rollout, run automated accessibility checks on the wrapper, keyboard-only setup/backup/restore, 200% zoom at 1366×768, forced-colors, reduced motion, and a manual screen-reader pass. Document known failures rather than hiding them.
